# Sheep Escape 게임 개발 과정

이 문서는 Sheep Escape 게임이 단계별로 어떻게 구상되고 구현되었는지를 설명합니다.

## 1. 기본 씬 설정

### 1.1 배경과 바닥 생성
게임 플레이에 필요한 기본 환경을 구성합니다. Unity 씬에 배경 이미지와 바닥을 배치합니다.

---

## 2. 양(플레이어) 구현

### 2.1 양을 씬에 추가
양(Sheep) 캐릭터를 게임 씬에 배치합니다.

### 2.2 양은 좌우로 키보드로 이동할 수 있어야 한다
양이 좌우 방향키 또는 A/D 키를 사용하여 좌우로 이동할 수 있도록 구현합니다.

### 2.3 양은 카메라 경계 내에서만 이동하도록 제한
양이 화면 밖으로 나가지 않도록 카메라 경계 내에서만 이동하도록 제한합니다.

### 2.4 양의 스프라이트 방향 전환
이동 방향에 따라 양의 스프라이트를 뒤집어 이동 방향을 시각적으로 표현합니다.

### 2.5 양은 충돌 감지가 필요하다
양과 불덩어리 간의 충돌을 감지하기 위해 양에 충돌 감지 시스템이 필요합니다.

---

## 3. 불덩어리 구현

### 3.1 불덩어리 오브젝트 생성
떨어지는 불덩어리 오브젝트를 생성하고 프리팹으로 만들어 재사용 가능하도록 설정합니다.

> **프리팹(Prefab)이란?**  
> 프리팹은 Unity에서 게임 오브젝트의 재사용 가능한 템플릿입니다. 한 번 설정한 오브젝트를 프리팹으로 만들면, 같은 오브젝트를 여러 번 쉽게 생성할 수 있습니다. 불덩어리처럼 게임 중에 여러 개가 필요한 오브젝트는 프리팹으로 만들어서 `Instantiate()` 메서드로 동적으로 생성합니다.

### 3.2 불덩어리는 아래로 떨어져야 한다
불덩어리가 중력처럼 아래로 떨어지도록 구현합니다.

### 3.3 불덩어리는 화면 밖으로 나가면 자동 제거
불덩어리가 화면 하단을 벗어나면 메모리에서 자동으로 제거합니다.

### 3.4 불덩어리 자동 생성 시스템
게임 시작 시 자동으로 불덩어리를 시간차로 여러 개 생성합니다.

### 3.5 불덩어리는 카메라 범위 기반으로 스폰 위치가 계산된다
화면 전체 너비에 걸쳐 불덩어리를 랜덤하게 생성하기 위해 카메라 범위를 기반으로 스폰 위치를 계산합니다.

### 3.6 불덩어리는 충돌 감지가 필요하다
불덩어리와 양 간의 충돌을 감지하기 위해 불덩어리에 충돌 감지 시스템이 필요합니다.

---

## 4. 양과 불덩어리의 상호작용

### 4.1 양과 불덩어리 충돌 감지
불덩어리가 양에 닿았는지 감지합니다.

### 4.2 중복 충돌 방지
한 번의 충돌로 여러 번 게임 오버가 발생하지 않도록 중복 충돌을 방지합니다.

### 4.3 게임 오버 처리
충돌 시 게임 상태를 게임 오버로 전환하고 시간을 정지시킵니다.

---

## 5. 최적화 및 개선

### 5.1 중복 생성 방지
씬이 다시 로드되어도 불덩어리가 중복 생성되지 않도록 방지합니다.

### 5.2 기존 불덩어리 정리
게임 시작 시 씬에 남아있는 불덩어리를 자동으로 제거합니다.

### 5.3 게임 오버 상태 체크
게임 오버 시 모든 움직임과 입력을 무시하도록 체크합니다.

### 5.4 코드 구조화 및 주석
코드 가독성 및 유지보수성을 향상시키기 위해 구조화하고 주석을 추가합니다.

---
